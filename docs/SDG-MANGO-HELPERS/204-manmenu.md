# manmenu — Interactive Config Browser

**What this is for:** Not sure what a configuration option does? This tool shows each line of your config files with an explanation of what it controls.

Two-pane interactive menu for browsing mangoWM config lines with live explanations and documentation links.

## Usage

```
manmenu
```

Loops infinitely — press `Esc` or `Ctrl+C` to exit.

## Two-Pane Workflow

### Step 1: Select a config file

Lists files from `~/.config/mango/` with `bat` previews. Pick a file to inspect.

### Step 2: Select a config line

Shows each line from the chosen file in the left pane. The right pane (lower 50%) shows an explanation generated by `checker.sh`.

Press `Alt+b` to open the config key's documentation URL in Firefox via `browser.sh`.

## checker.sh — Key Explainer

Extracts the key name from a config line, looks it up in `options.list`, and provides detailed help for several categories:

| Category | Covers |
|----------|--------|
| `bind` | bindl, binds, bindr, bindp modifiers |
| `monitor` | Selectors (name/make/model/serial), modifiers (width/height/refresh/x/y/scale/vrr/rr/custom) |
| `dwindle` | Binary tree split behavior |
| `master` | Master/tiled primary/secondaries concept |
| `scroller` | Infinite strip, scrolling, stacking |
| `tearing` | Tearing setup and window rules |
| `env` | Environment variable propagation |
| `animations` | Types, durations, bezier curves |
| `bezier` | Bezier curve parameters (x1,y1,x2,y2), common curve types |
| `windowrules` | Filters, state modifiers, geometry, visuals, layout, animation, terminal, global |
| `tagrules` | Tag filters (id, monitor, make, model, serial) and modifiers |
| `layerrules` | Layer shell surface rules |
| `scratchpad` | Scratchpad behavior, named scratchpads |
| `overview` | Overview mode, hot corner, interaction |
| `autostart` | exec-once vs exec, delayed starts |

For `bind` lines, the key is further parsed into `MOD,KEY,<dispatcher>` format. Variants `bindl` (locked), `binds` (submap), `bindr` (released), `bindp` (pass) are normalized for lookup.

## browser.sh — Docs Opener

Parses the same key from the config line, looks up its documentation URL in `options.list` (field 4), and opens it in Firefox with:

```
firefox --new-tab <URL>
```

All URLs point to `https://mangowm.github.io/docs/...`

## options.list Backing Database

The 232-entry `options.list` at `~/.local/SDG-MANGO-CONF/options.list` powers both `checker.sh` and `browser.sh`. See `06-options-list.md` for format details.
